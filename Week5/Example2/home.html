Example 2 - XSS

XSS stands for Cross-Site Scripting. It is a type of security vulnerability typically found in web applications, where an attacker injects malicious scripts into content that is served to other users. These scripts can run in the victim's browser, potentially leading to data theft, session hijacking, and other malicious activities.

In XSS attacks, attackers exploit the lack of proper input validation or sanitization on a website, allowing them to inject malicious code (usually JavaScript) that gets executed when users view the affected web page.

Modern browsers, including Google Chrome, have built-in security mechanisms like Content Security Policy (CSP) and browser-specific XSS filters that help prevent basic Cross-Site Scripting (XSS) attacks. These mechanisms may automatically block or sanitize certain types of XSS.

However, if you'd like to see how the attack works for learning purposes, you can disable some of these protections (though it's not recommended to do this in real-world scenarios).

Try the following string as input in the below form: 

   <p onclick="alert('Hacked!')">Click me!</p>
